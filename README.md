# Character Group Bot

Telegram бот для управления персонажами и группами в ролевых играх.

## Функциональность

### Персонажи
- Создание персонажей с указанием имени, класса, уровня и описания
- Просмотр списка всех персонажей пользователя
- **Редактирование персонажей** - изменение любых параметров
- **Удаление персонажей** - деактивация ненужных персонажей

### Экипировка
- Добавление предметов экипировки к персонажу
- **Поддержка изображений** - можно прикрепить фото предмета
- Просмотр всей экипировки персонажа с изображениями
- Указание типа предмета и количества
- **Удаление предметов** - быстрое удаление ненужной экипировки

### Желания (Wishlist)
- Создание списка желаемых предметов для персонажа
- Приоритизация желаний (1-10)
- **Отметка выполненных желаний** - трекинг полученных предметов
- **Удаление желаний** - очистка списка от ненужного

### Группы
- Создание групп персонажей
- Добавление участников в группу (персонажей любых игроков)
- Просмотр всех участников группы
- Лидер группы может видеть экипировку и желания всех участников
- **Удаление участников** - исключение игроков из группы
- **Передача лидерства** - назначение нового лидера группы
- **Удаление группы** - полное удаление с исключением всех участников

**Как добавить игрока в группу:**
1. Игрок создает персонажа через `/createcharacter`
2. Игрок просматривает ID персонажа через `/mycharacters`
3. Игрок сообщает ID лидеру группы
4. Лидер добавляет участника: `/addmember ID_группы|ID_персонажа`

**Управление группой:**
- Только лидер может добавлять/удалять участников
- Только лидер может передать лидерство или удалить группу
- При передаче лидерства, старый лидер остается участником группы

## Установка и запуск

### Предварительные требования

- Java 21
- Maven 3.6+
- Telegram Bot Token (получить у [@BotFather](https://t.me/botfather))

### Конфигурация

1. Создайте файл `.env` или установите переменные окружения:

```bash
export BOT_NAME=YourBotName
export BOT_TOKEN=your_bot_token_from_botfather
```

2. Для продакшена настройте PostgreSQL в `application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/character_bot
    username: your_username
    password: your_password
```

### Сборка и запуск

```bash
# Сборка проекта
mvn clean package

# Запуск приложения
java -jar target/character-group-bot-1.0.0-SNAPSHOT.jar
```

Или через Maven:

```bash
mvn spring-boot:run
```

### Сборка портативных дистрибутивов

Проект поддерживает сборку standalone дистрибутивов для macOS и Windows с встроенным JRE. Это позволяет запускать приложение без предустановленной Java на целевой машине.

#### Быстрый старт

```bash
# Собрать дистрибутивы для всех платформ
./build-dist.sh

# Собрать только для macOS
./build-dist.sh macos

# Собрать только для Windows
./build-dist.sh windows
```

Дистрибутивы создаются в папке `target/dist/`.

#### Структура дистрибутива

```
character-group-bot-1.0.0-windows-x64/
├── app/
│   └── character-group-bot.jar    # Приложение
├── bin/
│   └── run.bat                    # Скрипт запуска
├── config/
│   └── .env.example               # Пример конфигурации
├── runtime/                       # Встроенный JRE 21
│   └── ...
└── README.txt                     # Инструкции
```

#### Запуск на целевой машине

**Windows:**
```batch
cd character-group-bot-1.0.0-windows-x64\bin
run.bat
```

**macOS:**
```bash
cd character-group-bot-1.0.0-macos-aarch64
./bin/run.sh
```

#### Альтернативная сборка через Maven

Можно собрать архив без JRE (для машин с установленной Java):

```bash
# Для macOS (создаст .tar.gz)
mvn clean package -Pdist-macos

# Для Windows (создаст .zip)
mvn clean package -Pdist-windows
```

#### Поддерживаемые платформы

| Платформа | Архитектура | Файл |
|-----------|-------------|------|
| macOS | Apple Silicon (arm64) | `*-macos-aarch64.tar.gz` |
| macOS | Intel (x64) | `*-macos-x64.tar.gz` |
| Windows | x64 | `*-windows-x64.zip` |

## Использование

### 🎮 Навигация с помощью кнопок

Бот теперь поддерживает **удобную навигацию с помощью интерактивных кнопок**!

При запуске команды `/start` появится главное меню с кнопками:
- **👤 Персонажи** - создание и управление персонажами
- **⚔️ Экипировка** - добавление предметов и просмотр снаряжения
- **⭐ Желания** - список желаемых предметов
- **👥 Группы** - создание групп и управление участниками
- **ℹ️ Помощь** - справочная информация

После выполнения любой команды появляются кнопки для быстрого перехода к другим разделам или возврата в главное меню.

### 📋 Интерактивные списки

**Список персонажей** (`/mycharacters`):
- Показывает кнопки в формате "Имя | ID"
- При нажатии на кнопку - полная информация о персонаже
- **Прямой просмотр:** кнопки "⚔️ Посмотреть экипировку" и "⭐ Посмотреть желания" показывают данные сразу
- Не нужно вводить команды - всё через кнопки!

**Список групп** (`/mygroups`):
- Показывает кнопки в формате "Название | ID"
- При нажатии на кнопку - информация о группе
- **Прямой просмотр:** кнопка "👥 Участники и экипировка" показывает всю информацию о группе сразу
- Быстрый доступ к управлению группой

**Это особенно удобно когда у вас много персонажей или групп!**

Все действия выполняются через удобные кнопки - вводить команды вручную не требуется.

### Основные команды

- `/start` - Открыть главное меню
- `/help` - Справка по командам

### Управление персонажами

**Через кнопки:** Главное меню → 👤 Персонажи

**Команды:**
- `/createcharacter имя|класс|уровень|описание` - Создать персонажа
  - Пример: `/createcharacter Арагорн|Воин|15|Следопыт Севера`
- `/mycharacters` - Список ваших персонажей
- `/editcharacter ID|имя|класс|уровень|описание` - Редактировать персонажа
  - Пример: `/editcharacter 1|Арагорн|Воин|20|Король Гондора`
- `/deletecharacter ID` - Удалить персонажа
  - Пример: `/deletecharacter 1`

### Управление экипировкой

**Через кнопки:** Главное меню → ⚔️ Экипировка

**Команды:**
- `/addequipment ID_персонажа|название|тип|количество|описание` - Добавить предмет
  - Текстовый вариант: `/addequipment 1|Меч Андуил|Оружие|1|Легендарный меч`
  - **С изображением**: Прикрепите фото и в подписи укажите ту же команду
    ```
    [Прикрепите фото]
    Подпись: /addequipment 1|Меч Андуил|Оружие|1|Легендарный меч
    ```
- `/myequipment ID_персонажа` - Просмотр экипировки (с фото если есть)
- `/deleteequipment ID_предмета` - Удалить предмет
  - Пример: `/deleteequipment 5`

### Управление желаниями

**Через кнопки:** Главное меню → ⭐ Желания

**Команды:**
- `/addwish ID_персонажа|название|тип|приоритет|заметки` - Добавить желание
  - Пример: `/addwish 1|Щит Элендила|Щит|10|Хочу получить`
  - Приоритет от 1 до 10 (10 - самый высокий)
- `/mywishes ID_персонажа` - Список желаний
- `/fulfillwish ID_желания` - Отметить желание выполненным
  - Пример: `/fulfillwish 3`
- `/deletewish ID_желания` - Удалить желание
  - Пример: `/deletewish 3`

### Управление группами

**Через кнопки:** Главное меню → 👥 Группы

**Команды:**
- `/creategroup название|описание` - Создать группу
  - Пример: `/creategroup Хранители Кольца|Братство`
- `/mygroups` - Ваши группы
- `/groupinfo ID_группы` - Информация о группе и участниках
- `/addmember ID_группы|ID_персонажа` - Добавить участника (любого игрока)
  - Пример: `/addmember 1|2`
  - ID персонажа можно узнать у игрока (команда `/mycharacters`)
- `/removemember ID_участника` - Удалить участника из группы
  - Пример: `/removemember 2`
- `/transferleader ID_группы|ID_участника` - Передать лидерство
  - Пример: `/transferleader 1|2`
  - ⚠️ После передачи вы перестанете быть лидером!
- `/deletegroup ID_группы` - Удалить группу
  - Пример: `/deletegroup 1`
  - ⚠️ Все участники будут исключены!

## База данных

Проект использует Liquibase для управления схемой БД. Миграции находятся в `src/main/resources/db/changelog/`.

### Таблицы:
- `users` - Пользователи Telegram
- `characters` - Персонажи
- `equipment` - Экипировка персонажей
- `wishlists` - Желания персонажей
- `groups` - Группы
- `group_members` - Участники групп

## Разработка

### Добавление новых команд

1. Создайте класс обработчика, реализующий интерфейс `CommandHandler`
2. Добавьте аннотацию `@Component`
3. Реализуйте методы `canHandle()`, `handle()` и `getCommand()`
4. Spring автоматически зарегистрирует обработчик

### Расширение функционала

- Добавьте новые entity в пакет `model`
- Создайте репозитории в пакете `repository`
- Реализуйте сервисы в пакете `service`
- Добавьте обработчики команд в пакет `telegram.handler`
